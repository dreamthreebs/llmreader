# LLM Paper Reader

用 Cursor Agent 精读物理/宇宙学论文的工作区。

---

## 30 秒上手

```
1. 给 Agent 一个 arXiv 链接（或把 LaTeX 源码放进 papers/ 目录）
2. 说「先读」→ Agent 自动生成 总纲 + 故事版 + 图表版
3. 对感兴趣的章节说「读 §2」→ 逐段精读
4. 不懂就追问 → Agent 把追问和解答写进笔记
```

---

## 口令速查

### 拿到新论文时

| 口令 | 做什么 | 示例 |
|------|--------|------|
| **先读** / **新论文** | 一次性生成三个总览文件：总纲 → 故事版 → 图表版 | `先读` |
| **写总纲** | 只写五问总纲 | `写总纲` |
| **写故事版** | 只写通俗叙事版 | `写故事版` |
| **写图表版** | 只写逐图解读版（含图片） | `写图表版` |

### 深入精读时

| 口令 | 做什么 | 示例 |
|------|--------|------|
| **读 §X** | 按 subsection 精读，每次一个 subsection | `读 §2` 从 §2.1 开始；`读 §2.3` 直接跳到 §2.3 |
| **下一段** | 继续读当前 section 的下一个 subsection | `下一段` |
| **读公式 (X)** | 精读某个公式：符号表 + 逐项解释 + 动机 | `读公式 (7)` |
| **鸟瞰 §X** | 快速概览某节的结构、假设、结论（不逐句） | `鸟瞰 §3` |

### 理解和思考时

| 口令 | 做什么 | 示例 |
|------|--------|------|
| **追问** | 对刚读的内容追问，Agent 把解答写进笔记 | `δ_x 为什么在中性区是 -1？` |
| **串联公式** | 把关键公式串联，梳理推导依赖链 | `串联公式 §2` |
| **回顾** | 总结最近读过内容的逻辑推进线 | `回顾` |
| **批判** | 审视假设和近似，找作者跳步的地方 | `批判 §2.3 的假设` |

### 质量控制

| 口令 | 做什么 | 示例 |
|------|--------|------|
| **检查 §X** | 对某节笔记做全面校验（见下方详细说明） | `检查 §1` |
| **检查总纲** | 校验总纲中的物理陈述和关键数字 | `检查总纲` |
| **检查图表版** | 校验图表版中的图片描述和物理解释 | `检查图表版` |

## 推荐工作流

### 快速通读（1 小时了解一篇论文）

```
给链接 → 「先读」
  ↓
读 总纲.md（5 min）→ 知道论文在解决什么问题、核心想法是什么
  ↓
读 故事版.md（20 min）→ 建立物理直觉
  ↓
读 图表版.md（30 min）→ 对着图理解每个论证
  ↓
觉得哪里不懂 → 追问 / 读 §X 深入
```

### 深入精读（需要真正掌握论文）

```
快速通读之后 →
  ↓
读 §2（核心方法和公式）→ 读 §3（数值结果）→ ...
  ↓
串联公式 → 梳理推导链
  ↓
批判 → 审视假设和近似
  ↓
检查 → 确保笔记准确
```

## 四层笔记体系

每篇论文产生四层文件，覆盖四个理解层次：

| 文件 | 覆盖层次 | 阅读时间 | 适合场景 |
|------|---------|---------|---------|
| **总纲.md** | 问题 → 想法 → 结论 | 5 min | 快速回忆；给别人讲论文；自检是否真的懂了 |
| **故事版.md** | 物理图景 | 20 min | 从零了解一篇论文；建立直觉 |
| **图表版.md** | 证据链条 | 30 min | 对着图理解论证；准备组会报告 |
| **§N.md** | 公式推导 + 追问 | 按需 | 深入某节细节；解决具体困惑 |

### 总纲回答五个问题

```
1. 问题是什么 — 一句话 + 领域背景 + 以前方法的不足
2. 核心想法   — "这个方法之所以有效，是因为 ______"
3. 技术实现   — 方法流水线 + 关键近似
4. 证据       — 最重要的 3 张图 + 关键数字
5. 结论与影响 — 核心结论 + 局限 + 开放问题
6.（进阶）   — 如果我是作者，思维链是什么
```

**自检方法**：如果总纲里的五个问题你都能不看笔记自己答出来，说明你真的懂了这篇论文。

## 校验系统

校验是保证笔记质量的关键。分三种场景：

### 精读笔记校验（`检查 §X`）

Agent 回到 LaTeX 源文件，对 `§X.md` 做六步检查：

| 步骤 | 检查什么 | 怎么判断 |
|------|---------|---------|
| 1. 原文完整性 | 每个段落的英文原文是否完整逐字复制 | 对照 .tex 文件 |
| 2. 翻译忠实度 | 翻译有无遗漏限定词、改变语气 | 逐句对照 |
| 3. 物理正确性 | 物理图景、因果关系、近似条件是否正确 | 对照原文 + Agent 判断 |
| 4. 公式准确性 | 符号定义、动机解释是否与原文一致 | 对照 .tex 公式 |
| 5. 来源标注 | [原文]/[重述]/[补充] 标记是否准确 | 逐条核对 |
| 6. 追问质量 | 追问解答是否物理正确、来源标注是否明确 | 对照原文 |

每项用 ✅ / ⚠️ / ❌ 标记。发现的问题**当场修正**，修正记录追加到笔记末尾的「校验记录」。

### 总纲校验（`检查总纲`）

| 检查什么 | 怎么判断 |
|---------|---------|
| "一句话问题"是否准确 | 对照 §1 Introduction |
| "核心想法"是否抓住了真正的 insight | 对照全文逻辑 |
| 关键数字是否正确 | 对照原文具体数值 |
| 结论/局限是否遗漏 | 对照 Summary section |

### 图表版校验（`检查图表版`）

| 检查什么 | 怎么判断 |
|---------|---------|
| 图的描述是否与 caption 一致 | 对照 figures.tex |
| "怎么看"是否正确 | 对照正文对图的引用 |
| 物理解释的 [原文]/[补充] 是否准确 | 对照正文 |
| 图与图之间的逻辑衔接是否正确 | 对照论文结构 |

## 关于图表版的图片

图表版（`图表版.md`）引用了论文中的图片。图片存在 `papers/` 目录下，而 `papers/` 不进 git（太大），所以 clone 仓库后图片位置会显示空白。

**不影响使用**：图表版的文字解说是完整自包含的，没有图也能看懂。

**想看图怎么办**：

```bash
# 1. 下载对应论文的 LaTeX 源码
#    arXiv 页面 → Other formats → Download source → 解压到 papers/<id>/

# 2. 把 PDF 图转成 PNG（macOS 示例）
cd papers/<id>/figures
for f in *.pdf; do magick "$f" -resize '800x800>' -quality 85 "${f%.pdf}.png"; done

# 3. 图表版的 Markdown 预览就能正常显示了
```

## 注意事项

- 优先给 **LaTeX 源码**（`.tex`），PDF 效果会打折扣
- 大多数 arXiv 论文可在页面点 "Other formats → Download source" 获取源码
- 也可以直接给 arXiv 链接，Agent 会自动下载源码
- `papers/` 已被 `.gitignore` 排除，不会提交到 git
- 所有笔记用中文书写，术语首次出现时括号注英文
- 公式用 LaTeX 格式（行内 `$...$`，独立 `$$...$$`），Cursor 的 Markdown 预览可正常渲染
