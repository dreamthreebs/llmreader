# 推荐工作流

## 三种阅读模式

根据需要选择不同粒度的阅读模式。

---

### 模式 A：逐段阅读 (Paragraph-by-Paragraph)

**适用场景**：论证密集、逻辑链条长的段落。

**操作方式**：

1. 复制一段原文（3–8 句），发给 Agent
2. Agent 输出完整的 7 个栏目
3. 你确认理解后，再发下一段
4. 每 3–5 段做一次小结，让 Agent 回顾这些段落之间的逻辑链

**示例提示词**：

```
请用论文阅读模式帮我逐段阅读以下内容：

[粘贴原文段落]
```

**追问提示词**：

```
请回顾刚才读过的 3 段，总结它们的逻辑推进线。
```

---

### 模式 B：逐式阅读 (Equation-by-Equation)

**适用场景**：推导密集的章节，公式之间有强依赖关系。

**操作方式**：

1. 给出一个公式 + 原文中紧邻的上下文说明
2. Agent 按公式处理规则输出 7 项
3. 如果公式依赖前面的公式，将前面的公式编号一起给出
4. 追问各项的物理含义或近似条件

**示例提示词**：

```
请用论文阅读模式帮我理解这个公式：

$$[公式]$$

原文上下文：
"[公式前后 1–2 句原文]"

相关前置公式：Eq. (X)：$$[前置公式]$$
```

**追问提示词**：

```
这个公式中 [某一项] 在物理上代表什么？它和 Eq. (X) 的关系是什么？
```

---

### 模式 C：整节阅读 (Section-Level)

**适用场景**：已经对内容有初步了解，想快速把握一个 section 的整体结构和论点推进。

**操作方式**：

1. 把整个 section 的文字和公式一起给 Agent
2. Agent 先给出 section 级别的鸟瞰总结（作用、关键假设、主要结论）
3. 然后对 section 内的每个段落/公式逐一展开完整的 7 个栏目
4. 最后给出该 section 在全文中的定位

**示例提示词**：

```
请用论文阅读模式帮我阅读以下整节内容。先给出整节概览，再逐段展开。

## Section X: [标题]

[粘贴整节内容]
```

---

## 典型阅读流程

阅读一篇完整论文的推荐步骤：

### 第一遍：结构扫描

```
请帮我梳理这篇论文的整体结构：各 section 标题、每节大致内容、全文的逻辑推进线。

[粘贴目录或 section 标题列表]
```

### 第二遍：重点精读

选择最核心的 section（通常是方法论和主要结果），用模式 A 或 B 精读。

### 第三遍：公式串联

```
请帮我把这篇论文中的关键公式串联起来，说明它们之间的推导/依赖关系，以及各自的物理含义。

关键公式列表：
- Eq. (1): $$...$$
- Eq. (5): $$...$$
- Eq. (12): $$...$$
```

### 第四遍：批判性阅读

```
请帮我审视这篇论文中的关键假设和近似。哪些假设是标准的？哪些是本文特有的？有没有地方作者跳步太大？
```

## 实用追问模板

| 场景 | 提示词 |
|------|--------|
| 不理解某句话 | 「第 X 句说的 '...' 具体是什么意思？」 |
| 公式某一项不清楚 | 「公式 (X) 右边第二项在物理上代表什么过程？」 |
| 想知道近似的影响 | 「如果去掉这里的 [某近似]，结果会怎么变？」 |
| 想对比不同论文 | 「这篇论文的处理方式和 [另一篇] 有什么不同？」 |
| 想确认理解是否正确 | 「我的理解是 [你的理解]，这和原文一致吗？」 |
| 想了解动机 | 「作者为什么在这里选择这种方法而不是 [替代方法]？原文有说明吗？」 |
